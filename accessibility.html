Accessibility

Key elements and player

Any company would want to ensure their digital content meets their costumer needs. If they start to develop a mobile web app, accessibility should be one of their top priorities. According to the guidelines of the WCAG, UAAG an ATAG, the key elements to consider consist of:


  1. Web content - Like for desktop websites, information on mobile websites should be structured and presented in a way that minimizes cognitive load for users.

  2. Web technologies - HTML 5 and CSS. Even a special team force has been initiated to support accessibility in HTML5.

  3. Authoring tools - software packages that developers can utilize to create web content. Assistive technology - WAI-ARIA supports accessibility for people who cannot use mouses or rely on screen readers. It focuses on interactive controls and provides content roles for identifying page structure areas to better support keyboard accessibility and interactivity

  4. Evaluation tools - Through a number of automated evaluation tools (TAW mobileOK Basic checker, ready.moby), developers can validate the overall accessibility of their created mobile website.

  5. Key player:
    • Application provider
    • Content aggregator
    • Service provider
    • Cell-and smartphone provider
    • Webhosting provider
    • System integrator
    • Security provider

Source: Mobile player, based on: Durst, 2012
