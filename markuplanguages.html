'Mobilyzing' your website

Mobile specific HTML, CSS

Based on the mobile web opportunities provided on this website, businesses might now consider how to ‘mobilize’ their websites. For developers, this battle is still in progress but here are some HTML and CSS codes to think about to go mobile:


  1. HTML
    • Viewport tag: to properly fit the content to the screen: Example:
    • Device specific tags; examples:
    • CSS
    • Position: fixed: For Android 2.2+ and iOS 5+
    • Overflow: scroll: iOS5+
    • Media queries: to target specific features (screen, width, orientation, resolution) with CSS itself. The first method has the advantage that it does not create an additional HTTP request (good for performance), example:
    • Second method example:
    • Miscellaneous CSS
    • Webkit-user-select: none; - prevent the user from selecting text (also works on desktop WebKit)
    • Webkit-touch-callout: none; - prevent the callout toolbar from appearing when a user touches and holds an element such as an anchor tag.
