Web app vs native app

A comparison

Enterprises are not limited to develop a mobile web app in order to maintain customer relationships. In fact, for any chief information officer, the question arises whether to deploy a web app or a native app. Here is a summary of the differences:

"" Web app Native app
Avantages
  • All platforms are accessible with one application
  • Fast development with HTML, Java script and CSS
  • Compatability with all current mobile devices (smartphone and feature phone)
  • API access
  • Censorship-independence through deployment on owned server
  • Immediate availability after activation and modifications
  • Speed of response
  • Development support by the device manufacturer
  • High user acceptance
  • Easy to locate in app stores
Disadvantages
  • Effort to program
  • Not usable for all platforms / devices
  • Download effort
  • Updates
  • Slower than native apps
  • Limited offline availability
  • No standardized operating elements (buttons, text fields)
  • Adaption across other operated platforms necessary (different operating systems, browser, screen resolution)

Luckily, there is a happy medium called 'hybrid apps' that combines the advantages from both apps: They bring together the cost-effective, HTML5 platform independent characteristic with the distribution and operation advantages.
